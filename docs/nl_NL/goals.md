# Doelen

Een **doel** is een ontwikkelingsdoel voor een specifieke speler. Doelen registreren waar een speler aan werkt — ze vormen de verhalende tegenhanger van de numerieke evaluaties.

## Velden van een doel

- **Speler** — de eigenaar van het doel.
- **Titel** — een korte omschrijving ("Nauwkeurigheid zwakke voet verbeteren").
- **Omschrijving** — vrije tekst voor context, oefeningen, coachingnotities.
- **Status** — bijv. Niet gestart, In uitvoering, Behaald, Gestopt. Configureerbaar via Configuratie → Doelstatus.
- **Prioriteit** — bijv. Laag, Gemiddeld, Hoog. Configureerbaar via Configuratie → Doelprioriteit.
- **Streefdatum** — optioneel.
- **Aangemaakt door** — de coach die het doel heeft vastgesteld.

## Doelen aanmaken

Via de beheerpagina **Doelen** of de frontend voor coaches:

1. Kies een speler.
2. Vul titel, omschrijving, status en prioriteit in.
3. Optioneel een streefdatum.
4. Opslaan.

## Voortgang van doelen

Een coach kan de status en omschrijving in de loop der tijd bijwerken naarmate de speler eraan werkt. Het lijstfilter op status in **Doelen** laat je snel het lopende werk vinden.

## Zichtbaarheid

Spelers zien hun eigen doelen op het frontend-dashboard. Coaches zien de doelen van spelers in hun gecoachte teams. Beheerders zien alles.
