# i18n audit — May 2026

> Comprehensive sweep for hardcoded English strings (PHP, JS, stored DB content) that bypass the i18n layer and ship untranslated. Triggered by user-reported "wizard generator output still talks about sessions" — the audit confirms ~28 such leftovers across the codebase plus several other categories that warrant fixing.

## Scope

**In scope:**
- **Tier 1** — frontend views (`?tt_view=...`), anything end-users see directly
- **Tier 2** — wp-admin pages a non-developer interacts with (Configuration, Migrations, Authorization editor, Dashboard layouts, Lookups admin, Demo Data, Backup, Modules)
- REST controller error messages that surface to UI toasts
- Stored content seeded via migrations: lookup rows, role names, eval categories, trial tracks, formation templates, letter templates

**Out of scope:**
- Tier 3: debug logs, exception messages users never see, audit-log payload values
- Internal stable enum keys (`'active'` / `'open'` / `'completed'`) — keys, not labels; render-time translated via `LabelTranslator`
- DB column names, table names, REST URL slugs, capability names, permission codes
- Variable names containing "session" (only what's *output* matters)

## Headline numbers

| Category | Findings | Severity |
|---|---|---|

## A. Critical: ships English today (P0)

### A1. Inline JS in `FrontendConfigurationView.php`

Three lines emit hardcoded English from a `<script>` block injected into the rendered configuration page. **Sibling lines in the same file do this correctly** with `esc_js( __( ..., 'talenttrack' ) )` — the inconsistency is unintended.

- [src/Shared/Frontend/FrontendConfigurationView.php:394](src/Shared/Frontend/FrontendConfigurationView.php#L394) — `msg.textContent = first || ('Error ' + r.status)` — user sees "Error 500"
- [src/Shared/Frontend/FrontendConfigurationView.php:396](src/Shared/Frontend/FrontendConfigurationView.php#L396) — `'Network error.'` (line 431 does this correctly)
- [src/Shared/Frontend/FrontendConfigurationView.php:448](src/Shared/Frontend/FrontendConfigurationView.php#L448) — `window.alert('Error ' + r.status)`

**Fix:** wrap each in `esc_js( __( ..., 'talenttrack' ) )`. Half-day.

### A2. CSV import preview in `csv-import.js`

[assets/js/components/csv-import.js:102-110](assets/js/components/csv-import.js#L102) hardcodes:
- Status labels: `'Error' / 'Dupe' / 'OK'` (visible badges in the import preview table)
- `data-label` attributes for mobile responsive layout: `'Row' / 'Status' / 'Player' / 'DOB' / 'Team' / 'Notes'` — these render as column headers on mobile via `::before { content: attr(data-label); }`

No PHP `wp_localize_script` covers these — every Dutch admin sees them in English.

**Fix:** add the 9 strings to the localize call in `DashboardShortcode.php` and read from `TT.i18n.csv_*`.

### A3. `admin-methodology-media-picker.js` — no localize call exists

[assets/js/admin-methodology-media-picker.js:15-31](assets/js/admin-methodology-media-picker.js#L15) reads `TT_MethodologyMedia.modalTitle` / `.modalButton` etc. — but **no `wp_localize_script` call exists for that handle anywhere in `src/`**. The English fallbacks (`'Pick image'`, `'Add'`, `'Image'`) are the only thing that ever renders.

Bonus: line 31 hardcodes Dutch (`'Wordt toegevoegd bij opslaan'`) directly into JS innerHTML — non-Dutch sites see Dutch.

**Fix:** add the missing localize call alongside the existing `wp_enqueue_script` site for this picker.

### A4. Hardcoded `aria-label` attributes (accessibility + i18n)

- [assets/js/components/multitag.js:23](assets/js/components/multitag.js#L23) — close-pill `aria-label="Remove"`
- [assets/js/components/flash.js:66](assets/js/components/flash.js#L66) — dismiss link `aria-label="Dismiss"`

Screen readers read these aloud — Dutch users hear English announcements.

### A5. `CoachDashboardView` "Save Session" button

[src/Shared/Frontend/CoachDashboardView.php:257](src/Shared/Frontend/CoachDashboardView.php#L257) — `esc_html_e( 'Save Session', 'talenttrack' )`. Wrapped in `__()` but `'Save Session'` has no `.po` translation (the migration renamed only "activity"-shaped strings). Plus [DashboardShortcode.php:84](src/Shared/Frontend/DashboardShortcode.php#L84) localizes a JS key `save_session` with the same English msgid.

**Fix:** rename msgid to `'Save Activity'`, rename JS key to `save_activity`, update `assets/js/public.js:21,99`. Add `'Save Activity'` to `nl_NL.po` ("Activiteit opslaan").

## C. Hardcoded strings not wrapped at all (P1)

### C1. Demo-data Excel template builder

[src/Modules/DemoData/Excel/TemplateBuilder.php](src/Modules/DemoData/Excel/TemplateBuilder.php) lines 111, 114, 121, 134, 135, 136 — multiline instructions on the README sheet of the user-downloadable Excel template. Examples:

### C2. Wizard step lookup-name renders

Three wizard steps render raw `tt_lookups.name` values via `echo esc_html( $n )` instead of routing through `LookupTranslator::name()`:

- [src/Modules/Wizards/Evaluation/AttendanceStep.php:58](src/Modules/Wizards/Evaluation/AttendanceStep.php#L58) — attendance status `<th>` headers ("present", "late", "absent", "excused")

**Fix:** route through `LookupTranslator::name()` and feed via `QueryHelpers::get_lookups()` instead of `get_lookup_names()`. ~30 minutes per file.

## D. Stored content backfill (P1)

The `tt_lookups.meta.translations.nl_NL.name` pattern works — but the foundational seed (`0001_initial_schema.php`) never gained a Dutch backfill, and four other tables don't even have a translations column.

|---|---|---|---|

**Recommended fix:** one migration `0058_seed_lookup_translations_backfill.php` that writes `meta.translations.nl_NL.name` (and where applicable `.description`) for the rows above. ~50 individual rows. Idempotent (skip if already populated).

### D2. Tables missing a translations column entirely

These four tables hold user-visible content but have no JSON translations field:

- **`tt_roles`** (9 system roles seeded in `Activator::defaultRoleDefinitions()` lines 1467-1620) — Configuration → Authorization is currently 100% English on Dutch installs
- **`tt_functional_roles`** (5 system roles in Activator + 1 from `0048`) — Mentor + Head Coach + Assistant Coach + Manager + Physio + Other; surfaces in team-people picker, Configuration → Roles
- **`tt_eval_categories`** (4 mains + 21 subcategories from `0008` + 5 staff-tree categories from `0048`) — evaluation forms
- **`tt_trial_tracks`** (3 from `0036`: Standard / Scout / Goalkeeper) — directly substituted into parent letters via `{track_name}` placeholder, currently producing "deelgenomen aan onze Goalkeeper-stage" in Dutch letters

**Recommended fix:** two PRs.
1. Add `LabelTranslator::roleLabel()` / `::functionalRoleLabel()` / `::trialTrackName()` for the system rows (cheaper, works for stable enum keys; custom rows still need user-typed Dutch).
2. Or — the more SaaS-correct path — add `translations TEXT NULL` columns to the four tables and migrate `*Repository` reads to consult them. Larger change but unblocks third-party rows getting translations too.

**Recommendation:** start with `LabelTranslator` for system rows (ships immediately), revisit translations-column migration when the SaaS tenant story matures.

### D3. tt_eval_categories — no translation field at all

`tt_eval_categories` (added by `0008`, extended by `0048` for staff tree, extended by `0057` with a `meta` column) has no `translations` field. The 4 player categories + 21 subcategories + 5 staff categories ship English-only.

**Highest semantic value of any single fix in this audit** — evaluation forms are the most-used coach surface.

**Fix:** add `meta.translations.<locale>.label` support in `EvalCategoriesRepository`, ship a `0058_seed_eval_category_translations.php` migration writing nl_NL labels.

### D4. tt_formation_templates

[database/migrations/0032_team_development.php:160,177,194,211](database/migrations/0032_team_development.php#L160) — 4 formation names (Neutral/Possession/Counter/Press-heavy 4-3-3) + Activator-seeded copies. No translations field.

**Fix:** `LabelTranslator::formationName()` for the 4 system templates.

## E. Trial letter templates (P3)

[src/Modules/Trials/Letters/DefaultLetterTemplates.php:21-32](src/Modules/Trials/Letters/DefaultLetterTemplates.php#L21) — locale dispatcher does `strpos($locale,'nl')===0 ? Nl : En`. Any non-Dutch locale (`de_DE`, `fr_FR`, `es_ES`, `it_IT`, `pt_PT`) silently falls back to English.

The class docblock acknowledges this ("clubs in other languages use the editor surface to add their own"). Acceptable today — but flag for the multi-language epic (#0010).

**Bonus issue:** parent-facing letter `{track_name}` substitution feeds the untranslated `tt_trial_tracks.name` ("Goalkeeper") into a Dutch letter body — covered by D2.

## Open questions

1. **Bundle 2 (session→activity sweep)**: should the rename also touch internal stable identifiers — DB column `session_date`, REST routes `/sessions*`, PHP method names `create_session()` etc.? **Recommendation: leave them.** They're stable contracts; renaming is a separate breaking change with migration cost. Visible vocabulary alone fixes the user complaint.

2. **Bundle 4 (role labels)**: `LabelTranslator` extension vs. add `translations` column to `tt_roles` / `tt_functional_roles`? **Recommendation: `LabelTranslator` first** — ships immediately for the 9+6 system rows. Custom rows added by clubs still render their typed name. Revisit columnar approach with #0010 (multi-language) when DE/FR/ES support is needed.

3. **Bundle 5 (eval categories)**: schema change to `tt_eval_categories` is a real migration. **Confirm priority** — this is the most-used surface (every evaluation form) but also the most expensive single fix.

4. **Bundle 9 (methodology arrays)**: audit-only flagged this; would need a follow-up confirming whether these task strings surface to the UI at all (some might be internal CSV-only). **Defer if not user-visible.**

5. **CI audit script (`scripts/audit-no-legacy-sessions.php`)**: should be widened so future regressions trip the build. Bundle 2 fix.

## Process improvements (separate from fix work)

- Add a CI step that scans for `echo` / `print` statements with quoted strings outside `__()` in `src/Shared/Frontend/` + `src/Modules/*/Frontend/` + `src/Modules/*/Admin/`. False positives are tolerable; signal value high.
- Add a CI step that scans `assets/js/**/*.js` for `|| 'English'` fallback patterns and warns; cross-references against the matching PHP `wp_localize_script` call.
- For new tables shipping seeded reference data, require a `translations` JSON column (or `*_json` columns following `MultilingualField`) at schema-design time. Add to `docs/architecture.md` schema rules.
- Document the "i18n boundary" explicitly in `docs/contributing.md`: where strings cross from internal stable keys to user-visible labels, the translator is mandatory.
